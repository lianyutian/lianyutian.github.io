Pytest asyncio简介

Python asyncio 的概念

asyncio 是 Python 标准库中的一个模块，用于支持异步编程。它于 Python 3.4 引入，并在后续版本中不断完善（如 Python 3.5 引入 async/await 语法）。asyncio 的核心目标是处理 I/O 密集型任务（如网络请求、文件读写、数据库操作），允许程序在等待 I/O 操作时继续执行其他任务，从而实现并发而非并行（区别于多线程或多进程的并行执行）。

  • 关键概念：
    • **协程 (Coroutine)**：asyncio 的基础单位，使用 async def 定义的函数。协程是可暂停和恢复的函数，不阻塞主线程。它们通过 await 关键字在等待操作时让出控制权。
    • **事件循环 (Event Loop)**：asyncio 的核心引擎，类似于一个调度器。它负责监控多个协程的执行状态，当一个协程等待 I/O 时，切换到另一个就绪的协程。事件循环是单线程的，确保线程安全。
    • **任务 (Task)**：协程的封装形式，使用 asyncio.create_task() 创建。任务允许协程在事件循环中并发运行，并提供取消、等待等管理功能。
    • Future：表示一个异步操作的未来结果的对象，与任务类似，但更底层。Future 可以设置结果或异常，用于手动控制异步操作。
    • 异步上下文：包括队列 (asyncio.Queue)、锁 (asyncio.Lock)、信号量 (asyncio.Semaphore) 等，用于在异步环境中管理共享资源。

asyncio 不是真正的多线程，而是基于“协作式多任务”（cooperative multitasking），协程必须主动让出控制权（通过 await）。

Python asyncio 的原理

asyncio 的原理建立在“非阻塞 I/O”和“事件驱动”模型上，借鉴了 Node.js 的单线程事件循环，但更注重 Python 的语法友好性。

事件循环的工作机制

  • 事件循环是一个无限循环，不断检查注册的协程或任务的状态。
  • 当一个协程遇到 await 时，它会暂停执行，并将控制权返回给事件循环。事件循环则轮询其他任务。
  • 底层依赖于操作系统提供的非阻塞 I/O 接口（如 select、epoll、kqueue），这些接口允许程序检查多个文件描述符（sockets、files）的就绪状态，而不阻塞。
  • 示例流程：
    1. 创建事件循环：loop = asyncio.get_event_loop()（在 Python 3.7+ 中推荐使用 asyncio.run() 简化）。
    2. 注册协程：通过 asyncio.create_task(coro()) 或 loop.create_task()。
    3. 运行循环：asyncio.run(main())，直到所有任务完成或异常发生。
    4. 循环结束：处理清理，如关闭连接。

协程的执行原理

  • 协程是生成器（generator）的扩展，使用 yield from（旧语法）或 await。
  • 当协程被 await 时，它返回一个 Awaitable 对象（通常是 Future 或 Task），事件循环会等待其完成。
  • 协程栈：asyncio 维护一个协程调用栈，支持嵌套 await。
  • 异常处理：协程中的异常会传播到任务中，可通过 task.exception() 捕获。

与线程/进程的比较

  • 优势：低开销（无线程切换成本）、线程安全（单线程）、易于调试。
  • 劣势：不适合 CPU 密集型任务（会阻塞事件循环）。对于 CPU 任务，可结合 asyncio.to_thread()（Python 3.9+）或多进程。
  • GIL（Global Interpreter Lock）：asyncio 在单线程中运行，不受 GIL 影响，但如果结合多线程，仍需注意。

内部实现细节

  • asyncio 使用 selectors 模块抽象底层 I/O 多路复用。
  • transport 和 protocol：用于网络编程，如 loop.create_connection() 返回 transport（低级 I/O）和 protocol（数据处理）。
  • 兼容性：asyncio 支持与其他异步库集成，如 aiohttp（HTTP 客户端/服务器）、aiomysql 等。

基本用法

运行协程

要运行一个协程，你可以使用 asyncio.run() 函数。它会创建一个事件循环，并运行指定的协程。

并发执行多个任务

你可以使用 asyncio.gather() 函数并发执行多个协程，并等待它们全部完成。

超时控制

你可以使用 asyncio.wait_for() 函数为协程设置超时时间。如果协程在指定时间内未完成，将引发 asyncio.TimeoutError 异常。

Python asyncio 的应用

asyncio 广泛用于高并发场景，尤其在 Web 开发、爬虫和实时应用中。以下是典型应用和代码示例。

网络 I/O：异步 HTTP 请求

使用 aiohttp 库（需安装，但 asyncio 提供基础支持）。

应用：爬虫、API 调用。并发请求多个 URL，而不阻塞。

服务器应用：异步 Web 服务器

使用 asyncio 构建 echo 服务器。

应用：聊天服务器、WebSocket 服务。高并发连接处理。

文件 I/O 和定时任务

异步读文件：使用 aiofiles 库。
定时：asyncio.sleep() 用于非阻塞延迟。

应用：GUI 应用（如 Tkinter + asyncio）、定时器、日志处理。

其他高级应用

  • 数据库操作：使用 aiopg 或 asyncpg 进行异步 PostgreSQL 查询。
  • 消息队列：结合 aioamqp（AMQP）或 asyncio.Queue 实现生产者-消费者模式。
  • 微服务：在 FastAPI（基于 ASGI 和 asyncio）中构建异步 API。
  • 局限与扩展：对于阻塞代码，可用 loop.run_in_executor() 在线程池中运行。大型项目中，结合 uvloop（更快的事件循环实现）提升性能。
