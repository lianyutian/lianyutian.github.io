Docker

Docker 简介

  1. ‌什么是 Docker？

    Docker 是一个开源的应用容器引擎，而一个容器 containers 其实是一个虚拟化的独立的环境，让开发者可以打包他们的应用以及依赖包到一个可移植的容器中，然后发布到任何流行的 Linux 机器上，也可以实现虚拟化。容器是完全使用沙箱机制，相互之间不会有任何接口。

  2. ‌Docker 的应用场景

    • 开发环境标准化
    • 持续集成/持续部署（CI/CD）
    • 微服务架构
    • 快速扩展与弹性伸缩

Docker 核心概念

  1. ‌镜像（Image）‌
    • 只读模板，包含应用和运行环境
    • 分层存储结构
  2. ‌容器（Container）
    • 镜像的运行实例
    • 容器生命周期管理
  3. ‌仓库（Registry）
    • Docker Hub 公共仓库
    • 私有仓库搭建（如 Harbor）
  4. ‌Docker 引擎（Docker Engine）
    • 客户端-服务器架构
    • 组件：Docker Daemon、CLI、REST API

Docker 安装与配置

Docker 从 1.13 版本之后采用时间线的方式作为版本号，分为社区版 CE 和企业版 EE，社区版是免费提供给个人开发者和小型团体使用的，企业版会提供额外的收费服务，比如经过官方测试认证过的基础设施、容器、插件等。

Docker 基础操作

  1. ‌镜像操作‌

    • 拉取镜像：docker pull <image>
    • 查看镜像：docker images
    • 删除镜像：docker rmi <image>

  2. ‌容器操作

    • 启动容器：

      • 常用参数：-d（后台运行）、-p（端口映射）、-v（数据卷）

    • 查看容器：docker ps -a

    • 进入容器：docker exec -it <container> /bin/bash

    • 停止/删除容器：docker stop/rm <container>

  3. ‌日志与状态

    • docker logs <container>
    • docker stats

Dockerfile 与镜像构建

  1. ‌

    Dockerfile 语法

    ‌

    • 基础指令：FROM、RUN、COPY、CMD、ENTRYPOINT
    • 优化指令：WORKDIR、EXPOSE、ENV

  2. ‌

    构建自定义镜像

    ‌

    • docker build -t <image-name> .

  3. ‌

    镜像优化实践

    ‌

    • 选择轻量基础镜像（如 Alpine）

  1. ‌

    网络模式

    ‌

    • bridge（默认桥接网络）
