什么是 LLM？

  1. ‌定义‌
    大语言模型（LLM）是一种基于深度学习技术的人工智能模型，通过海量文本数据训练获得自然语言的理解与生成能力。其核心目标是模拟人类语言逻辑，处理文本推理、问答、翻译等任务‌。
  2. ‌核心特点‌
    • ‌大规模训练数据‌：依赖互联网公开文本、书籍、代码等多样化语料库，学习语言的复杂规律和上下文关联‌。
    • ‌无监督学习‌：多数 LLM 通过自监督学习（如预测掩码词、生成连贯文本）自动提取语言特征，无需人工标注数据‌。
    • ‌参数规模‌：模型参数量通常达到百亿甚至千亿级别，通过增加参数提升对复杂语义的捕捉能力‌。
  3. ‌应用场景‌
    • ‌智能对话系统‌：如 ChatGPT 等产品，提供拟人化交互体验‌。
    • ‌代码生成与补全‌：辅助开发者快速编写程序（如 GitHub Copilot）‌。
    • ‌多语言翻译‌：支持跨语言文本的高质量转换‌。
  4. ‌技术要点‌
    • ‌Token 处理‌：文本输入被拆分为最小单元（Token），模型通过分析 Token 序列生成结果‌。
    • ‌Temperature 参数‌：控制生成文本的随机性，值越高输出越多样化，值越低则越保守‌。
    • ‌超参数优化‌：包括学习率、批次大小等，直接影响模型训练效率和效果‌。

大模型基础单位 Token

基本定义‌

  • ‌最小处理单元‌：Token 是大型语言模型（LLM）处理文本时的基本单位，代表模型能理解和生成的最小意义片段‌。
  • ‌形式多样性‌：根据标记化方案的不同，Token 可以是单词（如 apple）、子词（如 trans-、form-er）或单个字符（如汉字“我”）‌。

核心特性‌

  1. ‌上下文依赖性‌
    • Token 的划分并非固定，同一词汇在不同上下文中可能被拆分为不同 Token。例如，“苹果”在“苹果派”中可能被整体视为一个 Token，而单独出现时也可能作为独立 Token 存在‌。
    • 这种灵活性使模型能更精准地捕捉语言中的组合语义（如复合词、短语）‌。
  2. ‌数值化表示‌
    • 文本输入后，Token 会被转换为数字标识符（如向量），供模型进行数学运算和模式识别‌。
    • 例如，句子 Hello, world! 会被拆分为 Hello 和 world 两个 Token，并分别映射为唯一数值‌。
  3. ‌处理限制‌
    • LLM 对输入/输出的 Token 数量有上限（如 4K、32K），这由模型架构的计算复杂度决定‌。
    • 英文中通常 1 Token ≈ 4 字符，但中文等表意文字可能以单字为 Token，导致相同文本长度下 Token 数量更多‌。

大模型局限性

知识依赖性与实时性局限‌

  1. ‌训练数据决定知识边界‌
    LLM 的知识完全依赖其训练数据集，无法覆盖未学习过的专业领域或非公开数据（如企业内部文档）‌。若训练数据存在错误或偏见，模型输出的答案可能不准确‌。
  2. ‌信息更新延迟‌
    模型无法实时获取最新信息（如 2025 年的政治事件或科技动态），导致回答时效性受限‌。

幻觉问题与理解偏差‌

  1. ‌虚构与事实混淆‌
    LLM 可能生成看似合理但完全错误的回答，例如虚构历史事件或科学结论‌。这种“幻觉”源于模型通过概率匹配生成文本，而非基于真实理解‌。
  2. ‌语义理解浅层化‌
    模型对输入的解读停留在模式匹配层面，缺乏深层语义理解。例如，它能生成关于“爱”的优美文字，但无法真正理解其情感内涵‌。
